同源策略是浏览器的一种安全特性，它限制了一个源上的文档和脚本如何与另一个源上的资源进行交互。浏览器可以同时从多个站点加载和显示资源。你可能同时打开多个页面，或者一个站点可以从不同的站点嵌入多个 iframe。如果这些资源之间的交互没有任何限制，并且某个脚本被攻击者攻击，那么该脚本可能会暴露用户浏览器中的所有内容。

## 1. 「源」的定义

以下是 MDN 上对 Origin（源）的定义：

> Web content's origin is defined by the scheme (protocol), hostname (domain), and port of the URL used to access it. Two objects have the same origin only when the scheme, hostname, and port all match.
>
> Some operations are restricted to same-origin content, and this restriction can be lifted using CORS.

Web 资源的「源」由用于访问它的 URL 的 scheme（协议）、主机名（域）和端口定义。只有当 scheme、主机名和端口都匹配时，两个对象才具有相同的「源」。某些操作仅限于同源的内容，可以使用 CORS 解除此限制。

简单来说，「源」指的就是网页等 Web 资源的来源，如果两个资源的 URL 的协议、主机名和端口是相同的，那么则认为它们是同源的，也就是说来源是相同的。

以下是一些示例：

```
# 同源，相同的 scheme (http), hostname (example.com)，不通的文件路径没有影响
http://example.com/app1/index.html
http://example.com/app2/index.html

# 同源，因为 HTTP 默认监听 80 端口，主机名不区分大小写
http://Example.com:80
http://example.com

# 不同源因为协议不同
http://example.com/app1
https://example.com/app2

# 不同源因为主机名不同
http://example.com
http://www.example.com
http://myapp.example.com

# 不同源因为端口不同
http://example.com
http://example.com:8080
```

## Site 的定义

非正式地，站点（Site）是一个网站（Website），它是一系列网页的集合，由同一域（domain）提供服务，并由单个组织维护。

站点由域名的可注册域部分确定，可注册域由公共后缀列表中的一条加上它之前的域名部分组成。 这意味着，例如 theguardian.co.uk、sussex.ac.uk 和 bookshop.org 都是可注册的域。
